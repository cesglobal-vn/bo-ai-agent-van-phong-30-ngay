## Cấu trúc lộ trình

30 ngày = **4 tuần × 5 buổi (Thứ 2 → Thứ 6) = 20 buổi học**, cuối tuần ôn luyện & hoàn thiện. Mỗi buổi làm 1 AI Agent và cho ra **1 kết quả dùng được ngay**.

| Tuần | Nhóm năng lực | Ngày | AI Agent trong tuần |
|---|---|---|---|
| **1** | Khởi động bộ AI Agent văn phòng | 01–05 | Viết thư · Tóm tắt tài liệu · Ghi chú họp · Lập kế hoạch · Viết báo cáo |
| **2** | Biến dữ liệu thành báo cáo có nhận định | 06–10 | Làm sạch dữ liệu · Tìm điểm đáng chú ý · Báo cáo chỉ số · Gợi ý biểu đồ · Tóm tắt cho quản lý |
| **3** | Chuẩn hóa giao tiếp, tài liệu & quy trình | 11–15 | Viết quy trình · Thông báo nội bộ · Phản hồi khách hàng · Đề cương trình bày · Kho hỏi đáp |
| **4** | Ghép thành Workspace cá nhân tự động | 16–20 | Thu thập thông tin · Phân loại nội dung · Theo dõi công việc · Báo cáo tuần · Workspace cá nhân |

## Mỗi file "Ngày-XX" gồm gì?

Mỗi buổi là một file bài học **tự chứa**, copy là học/dùng được ngay:

1. **🎯 Mục tiêu buổi học** & kết quả đầu ra.
2. **🧩 Bài toán thực tế** (mở đầu 5 phút).
3. **🗂️ AGENT.md** — đặc tả agent (dán làm System Prompt / Project Instructions).
4. **🧠 SKILL.md** — bản cài Skill Claude + mẫu câu lệnh nhanh (điền `[ ]` là chạy).
5. **👣 Diễn tiến 45 phút** — theo đúng nhịp 5–10–20–5–5 của chương trình.
6. **📝 Bài tập về nhà** — hoàn thiện dần bộ AI Agent cá nhân.
7. **✅ Tiêu chí đạt & Đo hiệu quả** trước/sau.

> Mỗi agent luôn chạy **3 giai đoạn**: ① Brainstorm ngữ cảnh (hỏi 4–6 câu) → ② Lập kế hoạch & xác nhận → ③ Triển khai. Học viên đa ngành nên **luôn để agent hỏi ngữ cảnh trước khi làm**.

## Liên kết với thư mục agent gốc

20 agent theo ngày là bản "thực chiến hằng ngày". Bản đặc tả chuẩn đầy đủ (12 mục) của 12 agent nền tảng nằm ở thư mục cha `Bo-AI-Agent/01..12`. Nhiều buổi trong Tuần 2–4 là **biến thể chuyên sâu** của các agent nền tảng (vd: "Làm sạch dữ liệu" thuộc họ Agent 06; "Tìm điểm đáng chú ý" thuộc họ Agent 07).

## Điều phối Sub-Agent theo ngành (áp dụng mọi buổi)

Học viên đa ngành, nên mỗi agent giữ **nền domain knowledge chung** và điều phối 5 sub-agent (🔎 Researcher bối cảnh → 🧠 Phương pháp nền → 🔗 Tổng hợp → 🛠️ Thực thi → ✅ Kiểm định) để đầu ra bám đúng bối cảnh khách hàng và **khớp form/template/dữ liệu bạn đưa lên**. Ví dụ: cùng buổi *Phân tích dữ liệu*, kế toán/sale/marketing nhận đầu ra khác nhau. Xem `../_CO-CHE-DIEU-PHOI-SUB-AGENT.md`. Mỗi file `BAI-HOC.md` có mục *🧭 Điều phối Sub-Agent theo ngành của bạn*.
